Feedback Page


Technical Description


The Feedback Page for the Educa platform is designed using HTML, CSS, and JavaScript to create a structured and user-friendly feedback form. It includes fields for name, email, feedback type, and message, using semantic HTML elements like fieldset and label for clarity. The page features a responsive layout with a color scheme that aligns with Educa’s branding. JavaScript validation provides real-time feedback on user input, enhancing usability. ARIA labels, high-contrast text, and keyboard navigation support ensure accessibility for all users. By integrating structured HTML, responsive CSS, and interactive JavaScript, the page offers a seamless and accessible feedback submission experience.


Accessibility


Accessibility in this page is ensured through semantic HTML elements like fieldset, legend, and label, improving screen reader compatibility. The high-contrast color scheme enhances readability, while responsive design ensures usability across devices. Keyboard navigation and ARIA labels further improve accessibility, making the form easy to navigate for all users.

Team Page


Technical Description


The Team Page for Educa is designed to showcase team members in a structured and visually engaging layout. Using CSS Grid and Flexbox, the page arranges member profiles in responsive cards, displaying their names, roles, and profile pictures. Each card includes social media links, allowing users to connect with team members. The design maintains a consistent color scheme and typography, aligning with the platform’s branding. Navigation is optimized with a clear structure and intuitive layout, ensuring a seamless browsing experience. The page is fully responsive, adapting to different screen sizes for mobile, tablet, and desktop users.


Accessibility


Accessibility in this page is ensured through semantic HTML elements like header, section, and article to improve screen reader navigation. Alt text for images and high-contrast colors enhance usability for visually impaired users. The keyboard-navigable layout ensures easy interaction, making the page inclusive and user-friendly across devices.

Content Page


Technical Description


The content page is designed to provide comprehensive information about Sustainable Development Goals (SDGs), with a particular focus on SDG 4: Quality Education. The page is structured using HTML and CSS, ensuring a clean and responsive layout. The navigation bar at the top allows users to easily access different sections of the website, while the sidebar provides quick links to specific parts of the content page. The main content area is divided into sections, each addressing different aspects of SDGs and SDG 4, such as their importance, targets, challenges, and global progress. The page also includes images to enhance visual appeal and understanding. A "Back to Top" button is provided for user convenience, allowing easy navigation to the top of the page. The footer contains essential links, contact information, and social media icons, along with links to the page editor profiles of the students involved in the project. This page is optimized for readability and accessibility, ensuring a seamless user experience.


Accessibility


Accessibility in the content page is addressed through semantic HTML elements like "header", "section", and "aside", which improve screen reader navigation. The use of descriptive "alt" text for images and clear headings ensures content is accessible to visually impaired users. Additionally, the responsive design and high-contrast color scheme enhance readability and usability across devices. The "Back to Top" button further improves navigation for all users.

Challenges and Lessons Learned


During the implementation of the Feedback Page, Team Page, and Content Page, several challenges were encountered. For the Feedback Page, ensuring proper form validation and user input handling required JavaScript-based validation to provide real-time error messages. Maintaining accessibility while using interactive elements like dropdowns and file uploads was another challenge, which was addressed through ARIA labels and keyboard navigation support. The Team Page faced issues with grid alignment and responsiveness, particularly in handling different screen sizes. Ensuring consistent profile card layouts and proper image scaling across devices required adjustments using CSS Grid and Flexbox. The Content Page required careful organization to structure large amounts of information effectively. Balancing readability with navigation complexity led to the use of semantic HTML and a sidebar for quick access to sections.
