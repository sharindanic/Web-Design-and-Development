User Profile Page

Technical Description

The User Profile page is designed to provide a general overview of a user's professional and personal data. The page is structured in a grid format to display content in categories such as profile data, goals, skills, frustrations, experience, and contact details.

HTML Structure The page uses <div> elements with specific classes like 'profile-container', 'card', and 'profile-section' to create a modular layout. Each section is encapsulated within a 'card' class, ensuring consistency in styling and spacing.


CSS Techniques
Grid Layout: `profile-container` uses `display: grid` with `grid-template-columns: repeat(2, 1fr)` to create a two-column grid that allows for balanced content distribution.
Flexbox: Within each card, Flexbox (`display: flex`) is used to align items vertically or horizontally, such as in the `profile-section` where the profile picture, name, and bio are centered.
Responsive Design: The page is designed to be responsive, with elements like the profile picture and input fields adjusting to different screen sizes.
Interactive Elements: Hover effects and CSS transitions are used on interactive elements like the upload of the profile picture and skill bars. An example is how the `skill-percentage` class uses `width` to set the skill level dynamically based on user input.

Key HTML Tags
<input> and <textarea>: Used for editable fields like name, bio, and goals, allowing users to input and update their information.
<img>: Used for displaying the profile picture and icons for goals, skills, and frustrations.
<ul> and <li>: Used to create lists for goals, frustrations, and experience.

Key CSS Properties
'grid-template-columns': Defines the two-column layout.
'border-radius': Adds rounded corners to cards and images for a modern look.
'box-shadow': Adds depth to cards, making them stand out.
'transition': Used for smooth hover effects on interactive elements.

Accessibility

*Semantic HTML: The use of semantic tags like <input>, <textarea>, and <img> with appropriate alt attributes ensures screen readers can interpret the content.
*Keyboard Navigation: All interactive elements are accessible via keyboard navigation.
*Contrast and Readability: The color scheme ensures sufficient contrast between text and background, improving readability for users with visual impairments.

Sitemap Page

Technical Description

The Educa Sitemap page is meant to provide users with an easy and interactive visual representation of the website structure, so they can navigate easily and efficiently. The page uses SVG to generate a live map with nodes representing significant sections of the website, such as Home, Volunteer, Profile, Feedback, and Team. All nodes are clickable, which allows users to access the respective page directly. The design also has a modern and minimalist appeal, with a hover effect added to the nodes to enhance interaction. The webpage is fully responsive, with no glitches on mobile devices or others. The inclusion of SVG ensures scalability and good image quality, while JavaScript contributes functionality to interactive nodes. Besides improving usability, it also adds an interactive aspect of browsing through the website's content.

Accessibility

*Interactive Elements: The nodes are made clickable using JavaScript, ensuring that users can navigate to different pages directly from the sitemap.
*Text Labels: Each node has a text label, making it clear what each node represents.
*Keyboard Navigation: The nodes are accessible via keyboard navigation, allowing users to interact with the sitemap without a mouse.

Content Page

Technical Description

The content page is designed to provide detailed information on the Sustainable Development Goals (SDGs) with special focus on SDG 4: Quality Education. The page is developed using HTML and CSS to be clean, well-organized, and responsive in structure. The primary content is divided into sections, each addressing key issues of SDG 4, such as its introduction, importance, goals, challenges, and achievements at the international level. The page also has a top navigation bar for easy access to other parts of the site, and a sidebar with quick links to some areas of the content page for enhanced user navigation. The headings and subheadings adopted provide a properly organized flow of information, and images and bullet points for improved visual appeal and readability. A inclusion of "Back to Top" to provide for ready ease in readily jumping to page top, an inclusion in footer that entails inclusion of links regarded as most needed, along with a presentation of social media and a link for accessing student participant editor page profiles are available. Accessibility and best-readibility use optimization through read-out.

Accessibility

Accessibility on the content page is prioritized through the use of semantic HTML elements such as <header>, <section>, and <aside>, which improve navigation for screen readers. All images include descriptive "alt" text to ensure accessibility for visually impaired users. The page features a high-contrast color scheme and responsive design, making it easy to read and navigate on various devices. The inclusion of a "Back to Top" button further enhances usability, allowing users to quickly return to the top of the page without excessive scrolling. These features ensure that the content is accessible and user-friendly for all audiences.

Challenges and Lessons Learned

While creating the Sitemap and User Profile pages, there were some problems encountered. For the Sitemap, creating an interactive and responsive SVG-based map required a lot of planning and testing. Creating the nodes clickable and visually appealing on different devices was a major problem, which was solved by using a combination of SVG, CSS, and JavaScript. The User Profile page was having trouble with handling a responsive grid layout and ensuring all interactive elements, such as the upload of the profile image and setting the skill percentages, were working well. This was addressed through the use of CSS Grid and Flexbox for layout and JavaScript for interactivity.

The other challenge was to make accessibility available for both pages. It was more difficult for the Sitemap to include text labels on nodes and make them screen-reader accessible and keyboard-navigable. For the User Profile, making all input fields and interactive elements accessible to both screen readers and keyboard users was imperative. These challenges made it obvious how much weight should be placed on mobile-first design, accessibility best practices, and thorough testing across diverse devices and browsers.
