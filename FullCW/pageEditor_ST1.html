Splash Page


Technical Description


The Splash Screen for the Educa platform is designed using HTML and CSS to provide an engaging and informative introduction, featuring a centered content section with a header, a mission statement focused on SDG 4: Quality Education, and a list of members. The design includes a full-screen background image, a semi-transparent white overlay for readability, and a loading animation to indicate an automatic redirection to the home page after 4 seconds. The color scheme, with teal, coral and dark blue accents, ensures visual consistency and alignment with the platform's branding. The responsive layout and clean typography enhance user experience, making the Splash Screen both functional and visually appealing.


Accessibility


Accessibility in this page is ensured through semantic HTML elements like "header", "h1" and "u1" which improve screen reader compatibility.The color scheme provides sufficient contrast for readability, and the responsive design ensures usability across devices. Additionally, the loading animation offers visual feedback during redirection, enhancing the experience for all users.

Volunteer Page


Technical Description


The Volunteer page for Educa is designed to provide users with an interactive and engaging platform to explore and join various volunteer programs. The page features a clean and organized layout with a dropdown menu for sorting programs by popularity, recency, ratings, or alphabetical order. Volunteer opportunities, such as tutoring, library support, mentoring, and workshops, are presented in scrollable cards with images, descriptions, and join buttons. User ratings and reviews are prominently displayed to build trust and showcase positive experiences. Additionally, a feedback section allows users to submit their reviews and ratings, enhancing community engagement. The page is styled with a consistent color scheme, responsive design, and intuitive navigation, ensuring a seamless user experience


Accessibility

Accessibility in this volunteer page is ensured through semantic HTML elements like "header", "nav", "footer" which improve screen reader compatibility. The use of descriptive "alt" text for images and clear labels for interactive elements like dropdowns and buttons enhances usability for visually impaired users. Additionally, the responsive design ensures the page is accessible across various devices and screen sizes.

Content Page


Technical Description


The content page is designed to provide comprehensive information about Sustainable Development Goals (SDGs), with a particular focus on SDG 4: Quality Education. The page is structured using HTML and CSS, ensuring a clean and responsive layout. The navigation bar at the top allows users to easily access different sections of the website, while the sidebar provides quick links to specific parts of the content page. The main content area is divided into sections, each addressing different aspects of SDGs and SDG 4, such as their importance, targets, challenges, and global progress. The page also includes images to enhance visual appeal and understanding. A "Back to Top" button is provided for user convenience, allowing easy navigation to the top of the page. The footer contains essential links, contact information, and social media icons, along with links to the page editor profiles of the students involved in the project. This page is optimized for readability and accessibility, ensuring a seamless user experience.


Accessibility


Accessibility in the content page is addressed through semantic HTML elements like "header", "section", and "aside", which improve screen reader navigation. The use of descriptive "alt" text for images and clear headings ensures content is accessible to visually impaired users. Additionally, the responsive design and high-contrast color scheme enhance readability and usability across devices. The "Back to Top" button further improves navigation for all users.

Challenges and Lesson learned


During the implementation of the Splash Screen, Volunteer Page, and Content Page, several challenges were encountered. For the Splash Screen, ensuring a smooth transition and accessibility during the redirection required careful use of meta tags and a loading animation. The Volunteer Page faced issues with responsive card layouts and dropdown functionality, which were resolved using flexbox and JavaScript. The Content Page struggled with organizing large amounts of information and ensuring accessibility, which was addressed through semantic HTML, ARIA labels, and a sticky sidebar for navigation. These challenges emphasized the importance of mobile-first design, accessibility, and thorough testing, providing valuable lessons in creating user-friendly and inclusive web pages
